The Strategy Pattern

Strategy lets the algorithm vary independently from clients that use it.
Gamma et al (1994: 315)

Why?

Alternative to inheritance

Change behaviour at runtime

Loose coupling

How?

Classic Strategy Pattern

Classic Strategy: Context

public class Context {
    private IStrategy _strategy;
    // Code to set _strategy
    public void ContextInterface() {
        _strategy.AlgorithmInterface();
    }
}

Classic Strategy: Strategy

public interface IStrategy {
    void AlgorithmInterface();
}

public class StrategyA : IStrategy {
    public void AlgorithmInterface () {
        // Implementation
    }
}

Functional Strategy

Functional Strategy: Context

var Context = function () {
    var strategy;
    // Code to set strategy
    this.contextInterface = function () {
        strategy();
    };
}

Functional Strategy: Strategy

// No interface needed
var strategyA = function () {
    // Implementation
}

Setting the strategy

Constructor: C#

public class Context {
    private IStrategy _strategy;
    public Context (IStrategy strategy) {
        _strategy = strategy;
    }
    public ContextInterface () {
        _strategy.AlgorithmInterface();
    }
}
Context context = new Context(new StrategyA());

Constructor: JavaScript

var Context = function (strategy) {
    var strategy = strategy;
    this.contextInterface = function () {
        strategy();
    };
}
var context = new Context(strategyA);
// or even anonymously
var context = new Context(function () {
    // Implementation
});
